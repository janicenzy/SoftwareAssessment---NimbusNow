Gathering Lightning Thoughts...

For the front page, I took inspiration and asked a peer (Anastasia) for assistance
with the clickable logo that transitions into the weather webpage. I learnt about timing the animation transition
from one webpage to another through css @keyframes. Additionally, I set the background to a
timelapse of evolving weather I got off youtube: credits to Martin (Timestorm Film).



As for the sign-up page, I created a secondary page that checks the form validation; takes in input from
the user and verifys if the boxes have been filled or not, returning either a completion or 'Please fill out the boxes'.
In an attempt to advance even further, I tried connecting an online SMTP server (as seen below in my javascript code).


The Storm

Making the weather page, at first I created a couple of storyboards to make a
rough draft/layout of how I want the website to appear to its users.



Starting out, I created the navigation bar at the top to which I struggled with making the navigation bar a block.
I tried utilising artifical intelligence to spot the error in my code and found that I hadn't set the colour and opacity of the bar,
creating an illusion of a navigation bar instead of a still block regardless of whether or not the user scrolls down or not;
so I set the position to fixed. Next over, I went on a research spree on learning about weather API's, how to fetch data and import the files.
The learning process was fairly easy compared to the steep hill of hardship when it came to the practical.
I started out my typing out the default screen of the weather site, however, with the particular API I'm using it would
only allow cities and not specific places/suburbs such as Cabramatta. I also used the weatherapi icons
and aligned it with the changing weather. The CSS component was fairly easy: I set the background, set the side panel with
the search button in addition to other weather-related information and nearby cities
(respective to our area - doesn't change depending on the user's location so that could be an extension task to complete).

Clearing Skies

Although the javascript component was challenging at first, I researched more to refine my scripting skills and broke it up into parts:

  1. Adding a click event that changes the cities info using fetchWeatherData();
  2. Search validation if the user clicks on the search button, an alert will
    show, if not then screen will reload to entered city and display its data
  3. Created a function that gets and returns the day, date and time
  4. Lastly, putting everything together under the fetchWeatherData(); function
    and called it at the end

Smooth Sailing!

After the javascript, I found a few errors with the overall appearance of the website so I changed the background images to
higher-quality captures and fixed the yellow overlay on my glassmorphism panel that holds the extra information + nearby cities.
Moreover, I opened attempted learning additional CSS page transitions - adding a better transntion for when the user clicks the logo
on the front page. Creating this website has not only allowed me ot refresh my skills in HTML, CSS and Javascript but also has helped me
gain confidence in my js coding, through all the tutorials I've read and watched online to the Grok Leanring and Code Combat tasks.
Although I found myself struggling more than enjoying myself, as I allowed myself to fail (mainly in linking the weatherapi) this task has
allowed me to enjoy the process of trial and error rather than staying behind the comfortable threshold of 'passable' and 'overachieving'.
Nontheless, NimbusNow grants you weather all day, everyday

Whether the weather is good or bad.
